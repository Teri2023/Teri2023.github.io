PCA:¶

What is PCA:¶

The concept of Principal Component Analysis (PCA) was introduced by mathematician Karl Pearson in 1901, who is considered one of the founders of mathematical statistics. PCA has since become a standard tool in data analysis for reducing complex data sets to simpler forms and extracting underlying meanings. According to Johnathan, PCA effectively simplifies complex data sets and reveals hidden patterns. In the original data set scatter plot below, data points are mixed, leading to high error bars in linear regression predictions. However, the original PCA scatter plot below, with its distinct blue, orange, and green data sets, shows clearer separation and condensation of data points. This reduction in redundancy allows for more accurate trend predictions and the creation of reliable prediction models.

How PCA work:¶

Step 1: We start with the X matrix, which has m rows and n columns. The first step is to remove any empty or "zero" values by subtracting the mean of each value in X. This normalization step ensures that our data is centered around the origin.

Step 2: Next, we calculate the covariance matrix. To do this, we multiply the X matrix by its transpose (X^T) and then divide by the number of columns (n). The transpose of X (X^T) changes the matrix from m rows and n columns to n rows and m columns. This step measures the strength of joint variability between variables and indicates the relationship between them.

Step 3: After calculating the covariance matrix, we find the eigenvalues (λ) and eigenvectors (V) for the X matrix. To find the eigenvalues, we use the formula Det(X - λI) = 0, where I is the identity matrix. Solving this equation gives us the eigenvalues. The eigenvectors are then found by setting up a system of equations using the X matrix, the eigenvectors, and the eigenvalues. The result is a set of orthonormal eigenvectors, with each eigenvector corresponding to one eigenvalue.

Step 4: Finally, we compute the PCA transformation. This is done by multiplying the original X matrix by the eigenvector matrix V. The resulting matrix, X_pca, represents our data in the new principal component space.

These steps transform the original data set into a simpler form, highlighting the most important patterns and relationships between variables.

SVD:¶

What is SVD:¶

Singular Value Decomposition (SVD) was discovered before Principal Component Analysis (PCA) and developed by multiple mathematicians over time. It was first identified by Eugenio Beltrami in 1873 and further refined through the 20th century, with significant advancements made using the Jacobi eigenvalue algorithm in 1958 and Kahan's algorithm in 1970. SVD is simpler than PCA and is widely used by data scientists to analyze complex, unstructured data sets by reducing large matrices into smaller, compact forms with high redundancy. When comparing the same dataset used for PCA, the SVD-transformed graph shows similar clustering and separation of data points, maintaining their positions and integrity while offering a more condensed representation. This makes SVD a valuable tool for data reduction and pattern recognition.

How SVD work:¶

Singular Value Decomposition (SVD) starts similarly to Principal Component Analysis (PCA) by normalizing the original matrix to remove zero values. The next step involves finding orthonormal sets U and V, which serve as matrix components. This process is similar to calculating vector components through orthogonal projection, where U and V represent the components of the original matrix. U is sized according to the original matrix, while V is a square matrix based on the number of columns. The third step involves calculating Sigma, which represents the variances within U and V. Finally, the main SVD formula is applied to decompose the matrix. Unlike PCA, which identifies relationships across the entire matrix, SVD focuses on the relationships between individual elements within the matrix, providing detailed insights into the data's structure.

Conclusion¶

When comparing SVD and PCA with the original dataset, we also used the Sklearn function for PCA. In the original dataset scatter plot, the orange and green data points trend from left to right, while the blue set shows distinct head and tail values. The PCA scatter plot shows similar PC2 and PC1 values but with different trending lines and positions for the orange, green, and blue sets. The SVD and Sklearn PCA plots yield similar results to the original dataset, indicating that PCA may have issues transforming the original dataset accurately, as seen in the discrepancies.

To conclude our research, we identified several pros and cons of Principal Component Analysis (PCA). On the positive side, PCA offers dimensionality reduction, noise reduction, outlier reduction, data compression, and data visualization. However, there are also notable drawbacks. PCA involves a complex algorithm, can be misleading if data scaling is not properly handled, may result in information loss, and struggles with non-linear relationships in the original dataset. Additionally, it can lead to overfitting. These issues are evident in the original PCA dataset plot previously discussed.
